## Eingebaut

### Video-Einbettungen

Bette YouTube-Videos mit der Komponente `VideoEmbed` ein. Um diese Komponente zu verwenden, benötigst du die ID des Videos, das du einbetten möchtest. Diese lässt sich einfach aus dem `v`-Parameter der Video-URL kopieren.

Die ID für `https://www.youtube.com/watch?v=Y_WnAEKoeA4` wäre beispielsweise `Y_WnAEKoeA4`.

Anschließend kannst du diese ID direkt an die Komponente übergeben, wie unten gezeigt.

```md
<VideoEmbed id="Y_WnAEKoeA4"/>
```

### Audio

Identisch mit dem HTML-Tag `audio`, aber `src` wird als Asset-Identifier einschließlich Dateiendung angegeben.

Beispiel:

```md
<!-- Spielt <root>/assets/examplemod/sounds/ambient.ogg ab -->
<Audio controls src="examplemod:sounds/ambient.ogg"/>
```

### Codeblöcke

Codeblöcke im Wiki sind mit Syntax-Highlighting erweitert und können optional auch einen Dateinamen anzeigen.

````md
<!-- Mit Sprache und Dateiname -->
```json my-file.json
{
    "id": "hello-world"
}
```

<!-- Nur Sprache -->
```json
{
    "id": "hello-world"
}
```
````

## Gestaltung

### `Callout`

Lenke die Aufmerksamkeit des Lesers mit Hervorhebungen auf bestimmte Informationen.

#### Eigenschaften

- `variant`
  - Kann einer der folgenden Werte sein: [`default`, `info`, `warning`, `danger`]<br/>(Standard: `default`)
- `title`
  - Benutzerdefinierter Titel, der angezeigt werden soll. Optional

#### Code

```jsx
<Callout>
    Dies ist eine Standard-Hervorhebung.
</Callout>

<Callout variant="info">
    Dies ist ein Info-Hervorhebung.
</Callout>
```

### `CodeTabs`

Das Wiki implementiert die [CodeTabs](https://codehike.org/docs/code/tabs)-Komponente von CodeHike, wie unten gezeigt.

Das kann nützlich sein, wenn du in Entwicklerdokumentation separate Varianten für Gradle-/Kotlin-DSL-Code anzeigen möchtest.

#### Code

````jsx
<CodeTabs>

```js !!tabs main.js
function lorem(ipsum, dolor = 1) {
    const sit = ipsum == null ? 0 : ipsum.sit
    dolor = sit - amet(dolor)
    return sit ? consectetur(ipsum) : []
}
```

```css !!tabs styles.css
body {
    margin: 0;
    padding: 0;
}
```

</CodeTabs>
````

## Spielinhalte

### `Asset`

<Callout type="important">
    Es wird empfohlen, die Kurzschreibweise der [Asset-Syntax](../folder_structure/pages#erweiterungen) zum Anzeigen von Assets zu verwenden.
</Callout>

Zeigt ein [Asset](../folder_structure/assets)-Bild an.

#### Eigenschaften

- `location`*
  - Identifier des Assets, das du anzeigen möchtest.

Alle Eigenschaften von `img` außer `src` werden ebenfalls akzeptiert.

#### Code

```jsx
<Asset width={64} height={64} location="minecraft:furnace" title="Ofen" />
```

## Icons

Die gesamte Sammlung der [Lucide Icons](https://lucide.dev/icons/) kann in Dokumentationsdateien über React-Komponenten verwendet werden.

Die Namen von Icon-Komponenten bestehen aus dem „Komponentennamen“ eines Icons, den du auf der Website finden oder durch eine fundierte Schätzung aus der Icon-ID ableiten kannst, indem du sie in PascalCase umwandelst (z. B. `align-vertical-justify-start` → `AlignVerticalJustifyStart`), und dem Suffix `Icon`.

Alle von [Lucide React](https://lucide.dev/guide/packages/lucide-react) unterstützten Eigenschaften sind ebenfalls verfügbar.

### Code

```jsx
<SquirrelIcon width={64} height={64} strokeWidth={1} />
```

### Ergebnis

<SquirrelIcon width={64} height={64} strokeWidth={1} className="mt-5" />

## Rezepte

<Callout type="important">
    Verwende auf Inhaltsseiten bevorzugt [Prefab-Komponenten](interactive#prefab), um Herstellung und Verwendung anzuzeigen.
</Callout>

### `ProjectRecipe`

Zeigt ein interaktives [Rezept](../folder_structure/game_data#rezepte) für ein einzelnes Item an. Dies ähnelt dem Drücken von `R` auf deiner Tastatur, wenn du Rezept-Viewer-Mods im Spiel verwendest.

#### Eigenschaften

- `id`*
  - Identifier des anzuzeigenden Rezepts.

### `RecipeUsage`

Zeigt eine Liste von Items an, die durch [Rezepte](../folder_structure/game_data#rezepte) unter Verwendung dieses Items hergestellt/erhalten werden können. Dies ähnelt dem Drücken von `U` auf deiner Tastatur, wenn du Rezept-Viewer-Mods im Spiel verwendest.

#### Eigenschaften

- `id`*
  - Identifier des Items, dessen Verwendung angezeigt werden soll.
