# Dokumentationsseiten

Dokumentationsseiten folgen einem Leitfaden-ähnlichen Layout *(genau wie die Dokumentation, die du gerade liest)*. Sie eignen sich am besten für Benutzerhandbücher, Tutorials und Entwickler-Dokumentationen deines Projekts.

Auf der Wiki-Webseite findest du diese unter dem Reiter „Dokumentation“ des jeweiligen Projekts.

## Platzierung

Dokumentationsseiten können beliebig benannt und innerhalb des Projekts verschachtelt werden. Sie müssen zwingend die Dateiendung `.mdx` verwenden.

Ordner, die du im Dateibaum erstellst, erscheinen im Wiki ebenfalls als Ordner in der Sidebar der Dokumentation.

## Erstellung

Dokumentationsseiten können die verfügbaren Frontmatter-Eigenschaften sowie alle nicht-interaktiven Komponenten nutzen.

Das Markdown-[Format](../pages#format) ist für alle Seitentypen identisch.

### Checkliste

Hier ist eine grundlegende Liste von Dingen, die du beim Schreiben von Dokumentationsseiten beachten solltest:

- [x] Lege [Frontmatter](../pages#metadata)-Eigenschaften nach Bedarf fest.

- [x] Erstelle eine [Ordner-Metadaten](../folder_meta)-Datei, um die Anzeigenamen der Seiten/Ordner in der Seitenleiste festzulegen.

- [x] Schau dir die verfügbaren [Komponenten](../../components/overview) an, die du einbinden kannst.

## Metadaten

Dokumentationsseiten erfordern keine zwingenden Frontmatter-Attribute, aber du kannst ein paar optionale Eigenschaften angeben.

Ein Anwendungsbeispiel mit Erklärungen findest du unten.

```mdx
---
# ID einer Asset-Ressource, die als Icon des dokumentierten Items angezeigt wird.
# Es sollte eine Mindestgröße von 128x128 Pixeln haben.
# (Optional)
icon: examplemod:item/fancy_generator

# Changelog (Versionsverlauf). Siehe übergeordnete Seite für Details.
# (Optional)
history: ...
---
```
