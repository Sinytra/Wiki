# Dokumentationsseiten

Dokumentationsseiten folgen einem leitfadenartigen Layout *(wie die Dokumentation, die du gerade liest)* und eignen sich am besten für Benutzerhandbücher, Tutorials und Entwicklerdokumentation für dein Projekt.

Auf der Wiki-Website befinden sie sich im Reiter „Dokumentation“ des jeweiligen Projekts.

## Platzierung

Dokumentationsseiten können beliebig benannt und innerhalb des Projekts verschachtelt werden. Sie müssen zwingend die Dateiendung `.mdx` verwenden.

Ordner, die im Dateibaum erstellt werden, erscheinen im Wiki ebenfalls als Ordner in der Dokumentations-Seitenleiste.

## Erstellung

Dokumentationsseiten können verfügbare Frontmatter-Eigenschaften und alle nicht interaktiven Komponenten verwenden.

Das Markdown-[Format](../pages#format) wird von allen Seitentypen gemeinsam genutzt.

### Checkliste

Hier ist eine grundlegende Liste von Dingen, die du beim Schreiben von Dokumentationsseiten beachten solltest:

- [x] Gib [Frontmatter](../pages#metadaten)-Eigenschaften nach Bedarf an.

<p/>

- [x] Erstelle eine [Ordner-Metadaten](../folder_meta)-Datei, um Anzeigenamen von Seiten/Ordnern in der Seitenleiste festzulegen.

<p/>

- [x] Sieh dir verfügbare [Komponenten](../../components/overview) an, die du einbinden kannst.

## Metadaten

Dokumentationsseiten haben keine erforderlichen Frontmatter-Attribute, aber du kannst einige optionale Eigenschaften angeben.

Ein Anwendungsbeispiel mit Erklärungen findest du unten.

```mdx
---
# Resource Location eines Assets, das als Icon des dokumentierten Items angezeigt wird.
# Das Asset sollte mindestens 128x128 px groß sein.
# (Optional)
icon: examplemod:item/fancy_generator

# Changelog. Details findest du auf der übergeordneten Seite.
# (Optional)
history: ...
---
```
