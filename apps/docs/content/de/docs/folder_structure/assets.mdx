# Assets

<Callout>
    Spare dir die Zeit, Item-Renderings manuell zu exportieren, indem du unser zugehöriges [**Gradle-Plugin**](../tools/gradle) verwendest. Es lässt sich in gängige Modloader-Toolchains integrieren und kann **gerenderte Assets** automatisch für dich generieren!
</Callout>

Assets werden verwendet, um Bilder von Items, Blöcken und anderen Projektinhalten auf der gesamten Website anzuzeigen.

Sie können als Teil von Crafting-Rezepten, Informationen in der Seitenleiste oder benutzerdefinierten Komponenten verwendet werden.

## Format

Assets werden über [Identifier](https://minecraft.wiki/w/Identifier) identifiziert und verwenden dieselbe Syntax wie im Spiel selbst. Es wird empfohlen, Asset-Namen an die IDs der zugehörigen Objekte im Spiel anzugleichen.

Gerenderte Projekt-Assets, einschließlich **Items und Blöcke**, sollten im Dokumentationsverzeichnis unter `assets/(modid)` abgelegt werden, wobei `modid` der Namespace jeder Textur ist, die du bereitstellen möchtest.

Standardmäßig verweisen Identifier auf `.png`-Dateien. Eine benutzerdefinierte Dateiendung kann an den Identifier angehängt werden, falls die Datei ein anderes Format hat.

Der Name der Datei kann frei gewählt werden.

### Beispiele

- `examplemod:overview` - Zeigt `<root>/assets/examplemod/overview.png`

<p/>

- `examplemod:item/generator` - Zeigt `<root>/assets/examplemod/item/generator.png`

<p/>

- `examplemod:gui/grinding.gif` - Zeigt `<root>/assets/examplemod/gui/grinding.gif`

<p/>

- `minecraft:crafting_table` oder `crafting_table` - Zeigt ein 3D-Rendering der Werkbank an.

## Verwendung

Assets werden dir im Wiki je nach Funktion für verschiedene Zwecke begegnen.

Die häufigsten Anwendungsfälle sind:

- Anzeigen von Bildern an beliebigen Stellen auf [Seiten](pages#erweiterungen).

- Bereitstellen von Icons für [Seiten](pages#metadaten) durch Angabe des `icon`-Frontmatter-Attributs.

- Bereitstellen von Icons für [Inhaltsseiten](pages/content_pages), die automatisch über das `id`-Frontmatter-Attribut der Seite als Asset-Adresse bezogen werden.

- Anzeigen von Items in [Rezepten](game_data#rezepte).

- Abspielen von [Audio](../components/common#audio)-Dateien.

## Hinweis zu Item-Assets

<Callout type="important">
    Kurzfassung: Lege Inventar-Renderings deiner Items unter `assets/(modid)/item` ab
</Callout>

Das Wiki verwendet eine feste Asset-Adresse, um Inventar-Renderings von Items an bestimmten Stellen anzuzeigen, z. B. beim Mapping von Item-IDs auf Assets.
Typische Verwendungsfälle sind Icons von Inhaltsseiten, Standard-Infobox-Layouts, Rezepte und vorgefertigte Komponenten für Rezeptverwendungen.

Bei Inhaltsseiten und der Infobox können Standard-Adressen im Frontmatter der Seite überschrieben werden.
Bei integrierten Komponenten muss das Wiki jedoch ein passendes Item-Asset finden und anzeigen können, **wenn es nur dessen ID kennt**.

Das Wiki sucht nach diesen Assets unter `<modid>:item/<item_id>`, was in deinem Projekt `<root>/assets/<modid>/item/<item_id>` entspricht.

Um beispielsweise `examplemod:generator` anzuzeigen, wird das Bild unter `<root>/assets/examplemod/item/generator.png` abgelegt.

## Erlaubte Dateitypen

Aus Sicherheitsgründen dürfen nur bestimmte Dateitypen in Wiki-Projekte eingebunden werden, unabhängig vom Speicherort.

Derzeit werden die folgenden Formate unterstützt:

- MDX-Dokumente: `.mdx`
- JSON-Daten: `.json`
- Bilder: `.png`, `.jpg`, `.jpeg`, `.webp`, `.gif`, `.svg`
- Audio: `.ogg`, `.mp3`, `.wav`

Wenn du eine Datei einbinden möchtest, die hier nicht aufgeführt ist, kannst du dich gerne an uns wenden.
