# Translations

Documentation can be localized into various languages supported by the wiki. This applies to all types of pages,
folder metadata as well as some game files.

## Placement

<FileTree>
    <FileTreeFolder name="<root>" defaultOpen>
        <FileTreeFolder name="docs" defaultOpen>

            <FileTreeFile name="introduction.mdx"/>
        </FileTreeFolder>

        <FileTreeFolder name="translated" defaultOpen>
            <FileTreeFolder name="de_de" defaultOpen comment="Follows same structure as root">
                <FileTreeFolder name="docs" defaultOpen>
                    <FileTreeFile name="_meta.json"/>
                    <FileTreeFile name="introduction.mdx"/>
                </FileTreeFolder>
            </FileTreeFolder>
        </FileTreeFolder>
    </FileTreeFolder>
</FileTree>

Localized files must be placed under `translated/<lang>` inside the documentation directory, where
`<lang>` is the code for your desired language as it appears in [this table](#supported-languages).

## Layout

The localized folder structure follows the same layout as the project root, with some exceptions to what files can be
placed under it.

Currently, the following content may be translated:

- [Documentation pages](pages/docs_pages)
- [Content pages](pages/content_pages)
- [Folder metadata](folder_meta)
- [Item properties](game_data#item-properties)

Translated content **takes precedence** over the default locale when being loaded.
However, untranslated files will still be shown even when the locale is non-default, therefore it is recommended to
translate all files whenever possible.
